Anchor

Constructor: Anchor(inner=None, role=None)

Name Type Description
inner obj the object (usually str or WebComponent ) wrapped by an anchor
role str he anchor role. This parameter is not used in a typical scenario. It usually set by other web-components, which is using the anchor to introduce a specific action.

An Anchor is used for creating a hyperlink to pages, files, email addresses, locations on the same page, or other web-resources defined by a URL address. The Bootwrap also uses the Anchor in conjunction with other web-components, for example, in creating a navigation menu.

Use the link(href) function to connect the Anchor to another web-resource defined by the href parameter. 

Anchor('Google Search').link('https://www.google.com/')

Select Input

Constructor: SelectInput(name, label, *options)

Name Type Description
label str the input label showing for user
name str the input name, used for getting an input value from the form
options tuple the set of options

A SelectInput is a component for selecting a specific options from the dropdown list. Use SelectInput.Option(name, value, disabled=False) class to define selector options. The name parameter defines what user sees in the options list. The value parameters defines what is set if user selects a specific options. The disabled parameters makes an option unavailable for selection.

The function as_radio() changes the selector look from the dropdown list to a set of radio buttons.

The as_disabled() function prevents user to select any options. 

options = [
  SelectInput.Option('One', 1),
  SelectInput.Option('Two', 2),
  SelectInput.Option('Three', 3, disabled=True)
]

SelectInput('Selector1', 'choice', 2, options)
SelectInput('Selector2', 'choice', 2, options).as_disabled()
SelectInput('Selector3', 'choice', 2, options).as_radio()

Text Input

Constructor: TextInput(name, label, value=None, placeholder=None)

Name Type Description
label str the input label showing for user
name str the input name, used for getting an input value from the form
value str the value to set
placeholder str the text displayed when no value is entered

A TextInput is a component for entering a text information.

The with_multirows(n) function expands the text input to n rows. Use this option if the user needs to enter a long piece of text.

The for_email() and for_password() functions configure the input for a specific type of information ( email and password accordingly).

The as_disabled() function prevents user from entering any values. 

# the text input
TextInput('Text1', 'text')
TextInput('Text2', 'text').with_multirows(3)
TextInput('Text3', 'text', placeholder='type here')
TextInput('Text4', 'text', 'Hello World!')
TextInput('Text5', 'text').as_disabled()

# the email input
TextInput('Email', 'email', 'my@email.com').for_email()

# the password input
TextInput('Password', 'password', '********').for_password()

Javascript

Constructor: Javascript(src=None, script=None, submap=None)

Name Type Description
src str the URL to load Javascript code
script str the Javascript code
submap dict The map with substitutions, binding the Javascript with Python objects

A Javascript is a web-component for inserting Javascript code into the HTML page. This can be done in two ways. You can specify an external Javascript file via a URL using the src parameter. Or you can define Javascript code using the script parameter. To make a connection between Javascript and Python objects use the submap parameter. submap is a dictionary where key is a name of a variable within Javascript to replace with value - Python object. If Python object is an instance of WebComponent the substitution process with place it's identifier into the Javascript 

# Javascript URL
Javascript('https://ajax...0/jquery.min.js')

# Javascript code
label = Text('Answer:').as_strong()
answer = Text('unknown').add_classes('mr-2 ml-2')
btn_yes = Button('Yes').as_success()
btn_no = Button('No').as_danger()

action = Javascript(
  script='''
    $("#btn_yes").on("click",function(){
      $("#answer").text("Yes");
    });
    $("#btn_no").on("click",function(){
      $("#answer").text("No");
    });
  ''',
  submap={
    'answer': answer,
    'btn_yes': btn_yes,
    'btn_no': btn_no
  }
)

Panel(
  label, answer, btn_yes, btn_no, action
)

List.Item

Constructor: List.Item(*items)

Name Type Description
title obj the first-level line to define an item name.
description obj the second-level line to show additional information about an item (default=None).
marker obj the object levelled with the item title and justified to the right element to present such information as a timestamp or badge (default=None).
icon obj Tte object justified to the left to present an item icon (default=None).
custom obj The object justified the right to present actions that can be applied to an item(default=None).

A List.Item is a web-component for constructing a List item. Its parameters are named to fit a general use-case. But you can adapt its behaviour to suit your application. For example, the custom parameter can be used to visualize a share price represented by an item or to provide action(s) with a share or both.

Please note, that share prices in the example are fictional and do not represent the real value of companies shares. 

List(
  List.Item(
      'Google (NASDAQ: GOOGL)',
      description= 'Price for a single Google share',
      icon=Icon('fab fa-google').as_primary(),
      marker="12:04:58 12/01/2021",
      custom=Text('$1000').as_heading(3).as_success()
  ),
  List.Item(
      'LinkedIn (NASDAQ: LNKD)',
      description= 'Price for a single LinkedIn share',
      icon=Icon('fab fa-linkedin').as_primary(),
      marker="12:04:58 12/01/2021",
      custom=Text('$837').as_heading(3).as_danger()
  ),
  List.Item(
      'Amazon (NASDAQ: AMZN)',
      description= 'Price for a single Amazon share',
      icon=Icon('fab fa-amazon').as_primary(),
      marker="12:04:58 12/01/2021",
      custom=Text('$1054').as_heading(3).as_danger()
  )
)

Navigation

Constructor: Navigation(*items)

Name Type Description
items tuple the navigation items.

A Navigation is a web-component for grouping other controls and providing an intuitive way to switch between these groups.

Use Navigation.Item(name, content, active=False) class to define navigation items. The name defines what user sees in the navigation list. The content defines what is displayed when a specific navigation item is selected. The active makes a navigation item content is visable for user (in other words it make this item selected).

By default Navigation is rendered in a form of anchors. To change the navigation style, apply the following functions as_tabs() , as_pills() and as_vertical() .

Use the as_tabs() function to make navigation looks like tabs.

Use the as_pills() function to make navigation looks like buttons.

Use the as_vertical() function to change navigation items orientation from horizontal to vertical. In the case of the vertical orientation, the navigation bar is displayed on the left and content on the right. 

Navigation(
  Navigation.Item('Chapter 1', 'Text 1', True),
  Navigation.Item('Chapter 2', 'Text 2'),
  Navigation.Item('Chapter 3', 'Text 3')
)

Navigation(
  Navigation.Item('Chapter 1', 'Text 1', True),
  Navigation.Item('Chapter 2', 'Text 2'),
  Navigation.Item('Chapter 3', 'Text 3')
).as_vertical()

Navigation(
  Navigation.Item('Chapter 1', 'Text 1', True),
  Navigation.Item('Chapter 2', 'Text 2'),
  Navigation.Item('Chapter 3', 'Text 3')
).as_tabs()

Navigation(
  Navigation.Item('Chapter 1', 'Text 1', True),
  Navigation.Item('Chapter 2', 'Text 2'),
  Navigation.Item('Chapter 3', 'Text 3')
).as_pills()

Navigation(
  Navigation.Item('Chapter 1', 'Text 1', True),
  Navigation.Item('Chapter 2', 'Text 2'),
  Navigation.Item('Chapter 3', 'Text 3')
).as_pills().as_vertical()
